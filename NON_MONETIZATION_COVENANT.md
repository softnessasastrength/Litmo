# The Non-Monetization Covenant

**This is currently a personal emotional containment system, not a public product.**

**Pillar Fit Check:** serves Authenticity Over Performance (Pillar 6) and
Softness Is Strength (Pillar 1) — refusing to extract value from someone's
vulnerability, stated once, plainly, instead of implied.

**Status:** A values pledge, not a contract. Read `LICENSE` (Mozilla Public
License 2.0) for the actual legal terms this repository is distributed
under — this document is what the founder additionally promises on top of
that license, in plain language, with no legal enforceability claimed
beyond what MPL 2.0 itself already provides.

---

## The pledge

Litmo — this repository, and anything forked from it under the same
license — will never:

- Charge money for access to any feature, chamber, or protocol.
- Sell, rent, license, or otherwise monetize any user's personal data —
  touch preferences, consent records, attachment patterns, private notes,
  reflections, letters, or any other category in
  `docs/DATA_CLASSIFICATION.md`.
- Add advertising, sponsorships, or third-party tracking/analytics SDKs.
- Introduce engagement-maximizing mechanics — streaks, notifications
  designed to pull you back in, growth metrics, or anything optimized for
  time-on-app rather than for the person using it.
- Require payment to exercise any GDPR-aligned right already described in
  `docs/GDPR.md` (access, export, erasure, rectification).

If this project is ever handed to someone else, sold, donated, or
inherited in any form, **this pledge transfers with it.** Whoever holds it
next inherits the same restriction, not a clean slate.

## Why this exists as its own document

`CONSTITUTION.md` already says this project isn't a shipping public
product and that complexity must be nameable as containment, not "good
engineering" performed for its own sake. `docs/REAL_PURPOSE.md` says the
same thing from the purpose side. This document exists because those are
statements about *what Litmo is*. This one is a promise about *what will
never be done to the people who use it* — including, if this is ever
forked, people who are not the founder and never agreed to anything by
using someone's fork of a repository.

## What this is not

- Not a substitute for reading `LICENSE` (MPL 2.0) — that's the actual
  legal instrument. This document has no legal force beyond what a public,
  dated, git-committed statement of intent carries on its own, which is
  real but is not the same as a signed contract.
- Not a claim that this project has lawyers, a compliance department, or
  certified anything. See `docs/GDPR.md` §11 and `docs/ISO27701.md` for
  what is and isn't actually claimed elsewhere.
- Not retroactively enforceable against a fork made *before* this document
  existed — but every fork made from this point forward inherits it as
  part of the repository.

## For forks (see `docs/FORK_YOUR_OWN_CATHEDRAL.md`)

If you fork this repository to build your own containment system: you are
not required to keep this exact document, but the MPL 2.0 license you
inherit means any changes you make to files that were already part of this
repo flow back upstream. If you choose to remove this pledge from your
fork, that's your right under the license — but know that you are choosing
to remove it, on purpose, not inheriting its absence by accident.
