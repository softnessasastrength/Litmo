# Litmo UX/UI Philosophy

**Reality check:** Litmo is currently a personal emotional containment system built by one founder for their own relationship, not a public product — there is no general user base today. "Users" below means the founder and their partner; treat these as personal design notes, not a company's product strategy.

## Fluid and Warm

Every interaction should feel fluid, calm, and intentional. Motion is part of the interface, not decoration.

Principles:
- Transitions should be smooth and continuous.
- Motion should communicate state changes, never distract.
- Warmth is conveyed through typography, spacing, color, and pacing.
- Interfaces should invite, never overwhelm.
- Friction is added only when it protects privacy, consent, or safety.
- Silence is a feature: avoid unnecessary prompts, banners, and interruptions.

Design question:
"Does this interaction feel like a calm conversation or a demanding application?"

Goal:
Users should leave Litmo feeling more grounded than when they opened it.
