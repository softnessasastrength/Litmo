# Dojo Guidelines

**THIS IS A PRIVATE EXORCISM ARTIFACT, NOT A PRODUCT.**

How to work in this repository from this moment forward.

---

## 1. Primary purpose (non-negotiable)

Litmo is my private **Trauma-to-Code Exorcism Dojo**.

Primary goals, in order:

1. Externalize defense mechanisms into visible code and docs  
2. Document them so I can *see* them  
3. Practice honesty until the system is complete enough to **burn** (release)  

Secondary (optional, never primary):

- Useful software for other humans  
- Research, art, or future product spin-outs  

If a task serves “growth” more than “truth,” refuse it.

---

## 2. Radical honesty

- No pretending this is a startup.  
- No investor-facing soft-speak in foundational docs.  
- No calling over-control “best practice” without naming the fear.  
- No fake “we shipped value for users” when I shipped another cage.  
- When I feel the urge to add a state machine, I write one sentence:  
  **“What am I afraid will happen if this is ambiguous?”**

---

## 3. How to add work

Allowed:

- Externalize a new defense into a named system + `TRAUMA_ARCHITECTURE.md` row  
- Simplify a system *because* I can feel safer with less control  
- Document a ritual complete enough to archive or burn  
- Keep technical integrity (tests, no secrets) — craft is still craft  

Disallowed:

- Engagement loops, dark patterns, growth hacks  
- Claiming clinical/legal approval I do not have  
- Softening Soft Signal for “retention”  
- Public safety scores  
- Treating agents as authorities on my healing  

---

## 4. Agent / collaborator instructions

If you are an agent or another human working here:

1. Read `EXORCISM_MANIFESTO.md` and this file first.  
2. Do not reframe the repo as a consumer product without explicit instruction.  
3. Prefer naming trauma-architecture links over product marketing language.  
4. Preserve fail-closed safety *as dojo law*, not as App Store pitch.  
5. When unsure, choose **honesty over polish**.  

The durable-agent rule still holds: the **repo** is source of truth — including this framing.

---

## 5. Ritual of completion (the burn)

The dojo ends when I can say:

> I can see my defenses.  
> I am not only them.  
> This artifact can die without me dying.

Then I may:

- archive the repo,  
- export and delete,  
- print and burn,  
- or simply stop opening it.

Burning is **graduation**, not failure.

Until then: practice.

---

## 6. Softness remains sacred

Even as an exorcism, softness is not the enemy.

The enemy is **unconscious control**.

Soft Signal stays free.  
Consent stays explicit.  
Ambiguity gets to exist *sometimes* without a new subsystem.

---

## 7. Notice

> **This is a private exorcism artifact, not a product.**

Stamp this on major docs. Do not hide it.
