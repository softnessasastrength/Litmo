# Product capability roadmap

The canonical implementation status lives in [`../ROADMAP.md`](../ROADMAP.md). Chapter 1 is a synthetic, local first playable spanning the whole conceptual journey so it can be evaluated before production architecture is attempted.

The longer-term capability sequence remains:

1. Touch Language onboarding
2. Body-zone consent setup
3. Discovery and session requests
4. Mutual Consent Snapshot confirmation
5. Active session and Soft Signal realtime exit
6. Independent wrap-up and Trust Ledger
7. Security, moderation, accessibility, legal, and safeguarding reviews before any real-world pilot

The Chapter 1 versions of these capabilities are simulations, not completed production systems. No later chapter begins without founder approval.
