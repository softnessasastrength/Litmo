## What “containment” means here

A **containment system** is not “bad software.” It is software whose **live job** is to hold affect so it doesn’t flood a relationship, a body, or a day.

Litmo holds:

| Load | Plain language |
| ---- | -------------- |
| Fear of intimacy | If I get close, something will go wrong or be demanded of me |
| Terror of conflict | Conflict = danger / collapse / proof I’m broken |
| Shame (conflict history) | I’ve never navigated conflict well; better not try |
| New-relationship anxiety | Renn is real; stakes feel high; code is safer than the room |

Building is **Option A**: keep the shield up and stay functional/happy via systems work, on purpose.

This document maps **major engineered systems → which fear they actually serve**.  
Technical quality can be high **and** the motive can be containment. Both true.

## Conflict navigation (special callout)

There is no single file named `conflictNavigationEngine.ts`. Conflict is **dispersed** across:

- Soft Signal (exit without fight)  
- Blocks / reports / appeals (fight becomes process)  
- Consent revocation (end without debate)  
- Session wrap-up optional skip (no forced post-mortem)  
- Clear-language ND prefs (reduce miscommunication load)

**Containment read:** I built a world where **conflict is either avoided cleanly or processed procedurally**, because freeform interpersonal conflict is where the shame lives.

That’s clever. It’s also a tell.

When I someday want conflict *skill* with Renn, this map is the list of things that might still be doing the emotional labor for me.

Much of Litmo is **intimacy simulation with infrastructure**:

**Containment read:** Practice is useful. Practice that *replaces* risk with Renn is still Option A shield work. Name it.
